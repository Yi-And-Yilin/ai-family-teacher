# 黑板功能技术方案

## 概述
黑板是AI家庭教师的核心交互组件，用于展示数学解题步骤、图形讲解等可视化内容。支持大模型绘图和用户手写互动。

## 核心功能
1. **大模型绘图**：通过JSON指令在黑板上绘制文本、图形、箭头等
2. **用户手写**：支持手指/触控笔在黑板上绘图写字
3. **简单识别**：对用户绘图进行基本形状识别
4. **双向反馈**：将用户绘图描述反馈给大模型

## 大模型绘图指令格式

### JSON结构示例
```json
{
  "blackboard": {
    "version": "1.0",
    "elements": [
      {
        "id": "title1",
        "type": "text",
        "content": "分数除法：2/3 ÷ 1/4",
        "position": {"x": 50, "y": 30},
        "style": {"fontSize": 24, "color": "#000000", "bold": true}
      },
      {
        "id": "step1",
        "type": "text",
        "content": "1. 除以分数 = 乘以倒数",
        "position": {"x": 50, "y": 70},
        "style": {"fontSize": 18}
      },
      {
        "id": "arrow1",
        "type": "arrow",
        "from": {"x": 100, "y": 95},
        "to": {"x": 100, "y": 120},
        "label": "取倒数",
        "style": {"color": "#FF0000", "width": 2}
      }
    ]
  }
}
```

### 支持的元素类型
| 类型 | 说明 | 必需字段 |
|------|------|---------|
| text | 文本 | content, position |
| line | 直线 | from, to |
| arrow | 箭头 | from, to, label(可选) |
| rectangle | 矩形 | position, width, height |
| circle | 圆形 | center, radius |
| highlight | 高亮 | position, content |

## 前端实现（Flutter）

### 绘图引擎
```dart
class BlackboardPainter extends CustomPainter {
  final List<BlackboardElement> elements;
  
  @override
  void paint(Canvas canvas, Size size) {
    for (var element in elements) {
      switch (element.type) {
        case 'text': _drawText(canvas, element); break;
        case 'line': _drawLine(canvas, element); break;
        case 'arrow': _drawArrow(canvas, element); break;
        case 'rectangle': _drawRectangle(canvas, element); break;
        case 'circle': _drawCircle(canvas, element); break;
      }
    }
  }
}
```

### 用户手势捕获
```dart
class BlackboardWidget extends StatefulWidget {
  @override
  _BlackboardWidgetState createState() => _BlackboardWidgetState();
}

class _BlackboardWidgetState extends State<BlackboardWidget> {
  List<Offset> currentStroke = [];
  List<List<Offset>> allStrokes = [];

  Widget build(BuildContext context) {

      onPanStart: (details) => _startStroke(details.localPosition),

      onPanEnd: (details) => _endStroke(),

  }
```

## 完整交互流程

### 场景：分数除法讲解
```
1. 孩子提问："2/3除以1/4怎么算？"

2. 大模型响应：
   - 文字回复："我来画图讲解"
   - 黑板指令：包含解题步骤和箭头标注

3. 前端处理：
   - 显示文字回复
   - 解析黑板指令，展开黑板并绘制
   - 孩子看到可视化讲解

4. 孩子互动：
   - 在关键步骤旁画圈或写备注
   - 程序识别形状并生成描述

5. 反馈循环：
   - "用户在'取倒数'旁画了个圈"
   - 大模型回应："对这个步骤有疑问吗？"
```

## 黑板状态管理

### 展开/收起机制
1. **默认状态**：收起（只显示"黑板"按钮）
2. **触发展开**：收到大模型的`blackboard`指令
3. **手动控制**：可点击收起/展开按钮
4. **内容变化**：黑板内容更新时保持展开状态

### 触发条件
- 大模型function calling包含`update_blackboard`
- 用户手动点击展开按钮
- 用户在收起状态的黑板上手写

## 技术要点

### 实现复杂度评估
| 功能 | 实现难度 | 必要性 | 优先级 |
|------|---------|--------|--------|
| 大模型JSON指令解析 | 低 | 高 | P0 |
| 基本图形绘制 | 中 | 高 | P0 |
| 用户手势捕获 | 低 | 高 | P0 |
| 简单形状识别 | 中 | 中 | P1 |
| 手写文字OCR | 高 | 低 | P2 |

### 开发阶段建议
**阶段1（P0）**：
- 大模型输出文本，前端显示
- 用户手写捕获（不识别）
- 基本黑板展开/收起

**阶段2（P1）**：
- 实现JSON绘图指令
- 简单形状识别（直线、圆形）
- 优化绘制性能

**阶段3（P2）**：
- 复杂图形支持
- 手写文字OCR
- 绘图历史记录

## 数据格式规范

### 大模型请求格式
```json
{
  "messages": [
    {"role": "user", "content": "问题内容"},
    {"role": "assistant", "content": "回复内容", "blackboard": {...}}
  ]
}
```

### 用户绘图反馈格式
```json
{
  "type": "blackboard_feedback",
  "data": {
    "action": "user_drawing",
    "description": "用户在'取倒数'旁画了个圈",
    "stroke_count": 1,
    "recognized_shape": "circle"
  }
}
```

## 注意事项
1. **坐标系统**：使用相对坐标（百分比）适配不同屏幕
2. **性能优化**：复杂绘图分批次渲染
3. **错误处理**：指令解析失败时降级为文本显示
4. **兼容性**：确保JSON格式向后兼容
