状态 A (DEFAULT_CHAT)： 收到任何字符，直接扔进聊天框。

状态 B (BOARD_MODE)： 一旦某一行以 B: 开头，进入黑板模式。该行后续内容全部送往黑板。

自动重置： 当遇到下一个换行符 \n 时，状态机自动重置回 DEFAULT_CHAT。

对其他场景无感： 其他场景没有 B:，状态机永远停留在 DEFAULT_CHAT，你的程序完全不需要改动。

模型负担轻： 模型只需要在写黑板那一行记得加 B:，写完一行回车，它就自动“退回”到聊天模式了。

1. 复杂数学公式：Markdown + LaTeX（绝对的行业标准）

如何运作：
你可以要求大语言模型在输出数学公式时，使用 LaTeX 语法，并用 $（行内公式）或 $$（独立块公式）包裹起来。

前端实现建议：

2. 特定格式与动画：原生 Markdown 的局限与突破

方案 A：Markdown 混合 HTML / CSS Class (最简单直接)
标准 Markdown 引擎通常允许内嵌 HTML 标签。你可以预先在前端写好带有动画效果的 CSS 类（比如 fade-in, highlight-pop）。

Plaintext

N: <span class="highlight-yellow">请记住这个重要推论！</span>
前端的 Markdown 引擎渲染这些 HTML 时，你的本地 CSS 动画就会自动触发。

方案 B：使用自定义指令 / MDX (更现代、更强大)
如果你想把组件做得更复杂（比如不仅仅是动画，还包含一个可以点击的交互式几何图形），推荐使用类似 MDX (Markdown + JSX) 的思路，或者自定义 Markdown 插件。

3. 给你的“全前缀架构”的一点防坑建议
