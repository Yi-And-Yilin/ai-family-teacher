# 用例：学习管家 - 错题回访

## 场景描述
一周前小明做错了一道通分题，系统按照"艾宾浩斯遗忘曲线"主动推送回访，检验是否真正掌握。

## 参与角色
- **学生**：小明
- **AI家庭教师**：系统（主动发起）

## 前置条件
1. 一周前（2024-01-08）小明做错题：1/3 + 1/4 = ?
2. 当时AI讲解后小明表示理解
3. 系统记录该题进入错题本，标记"需回访"
4. 系统根据算法计算最佳回访时间：7天后

## 基本流程

### 步骤1：系统主动推送
**时间**：2024-01-15 17:00（放学后）

**系统推送通知**：
"小明，一周前那道通分题还记得吗？今天再试试看：1/3 + 1/4 = ?"

**小明看到通知后**：
- 点击通知进入app
- 看到回访题目界面

### 步骤2：回访测试
**系统界面**：
```
🕐 错题回访 #1
📅 原错题日期：2024-01-08
📝 题目：1/3 + 1/4 = ?
💡 上次错误：直接相加得2/7（忘记通分）

请作答：
[答案输入框] ______
```

**小明思考后输入**："7/12"

### 步骤3：系统反馈
**系统**："答案正确！1/3 + 1/4 = 4/12 + 3/12 = 7/12。"

**系统**："和一周前相比进步很大！当时你直接加了分子分母得到2/7，现在完全掌握了通分方法。"

### 步骤4：掌握度确认
**系统**："为了确认真正掌握，我们再试一道类似题：1/5 + 1/2 = ?"

**小明**："1/5 = 2/10，1/2 = 5/10，2/10 + 5/10 = 7/10。"

**系统**："完美！你不仅记住了答案，还掌握了方法。"

### 步骤5：状态更新
**系统更新记录**：
- 错题状态：从"需回访"改为"已掌握"
- 掌握确认：通过回访测试
- 移出高频回访队列
- 进入低频维护队列（30天后再测试）

### 步骤6：奖励与鼓励
**系统**："🎉 恭喜！你成功掌握了这个知识点！"
- "获得'错题终结者'积分：+50"
- "连续3次回访正确，解锁'长期记忆大师'徽章"
- "学习进度条更新：分数运算掌握度+5%"

## 备选流程

### 备选A：回访错误
如果小明再次做错：
**小明输入**："2/7"（和一周前同样的错误）

**系统**："看来这个方法还需要巩固。我们重新讲解："
1. "先找3和4的最小公倍数：12"
2. "1/3 = 4/12，1/4 = 3/12"
3. "4/12 + 3/12 = 7/12"

**系统**："这道题重新标记为'需重点回访'，3天后再测试。"

### 备选B：部分正确
如果小明答案正确但过程不对：
**小明输入**："0.583"（小数形式）

**系统**："答案数值正确，但分数题最好用分数形式。7/12 ≈ 0.5833。"

**系统**："记住：分数运算保持分数形式更准确，避免小数精度问题。"

## 后置条件
1. 错题掌握状态更新
2. 学习数据记录
3. 积分/徽章奖励发放
4. 安排下一次回访（如果需要）

## 业务规则
1. **回访时间算法**：
   - 第一次错：1天后回访
   - 第二次错：3天后回访
   - 第三次错：7天后回访
   - 正确后：30天后维护性回访

2. **状态流转**：
   - 新错题 → 需回访
   - 回访正确 → 已掌握
   - 回访错误 → 需重点回访
   - 连续3次正确 → 长期掌握

3. **奖励机制**：
   - 回访正确：基础积分
   - 连续正确：额外奖励
   - 掌握难点：徽章奖励
