# 小书童 (AI Family Teacher) - 全功能行动计划

这个计划整合了所有开发任务，不再分阶段，目标是直接构建一个功能完整、AI驱动的家庭教师应用。

## 1. 核心 AI 与 通信架构 (底层基座)
- [ ] **完善 LLM 底层连接**：验证 Ollama 服务稳定性，并集成 DeepSeek API 作为备用，实现服务自动切换与重试。
- [ ] **建立消息存储与序列化格式**：扩展 `Conversation` 模型，定义统一的 JSON Schema，支持完整对话历史的分页加载与上下文管理。
- [ ] **实现 Chat Query Stream 流**：重构 `ai_service.dart` 以支持流式响应，并在 UI 端实现实时打字机输出效果。
- [ ] **构建 Agent 架构**：开发 `AgentManager`，定义“答疑助手”、“作业批改”、“笔记整理”等不同角色的 Agent，实现任务路由。
- [ ] **实现 Function Calling 系统**：定义 AI 可调用的工具函数（如 `update_blackboard`, `save_note`, `check_homework`），使 AI 能直接操作应用组件。

## 2. 核心教学组件 (三大件)
- [ ] **黑板 (Blackboard)**：
    - 深色背景绘制引擎。
    - 支持 AI 自动写入指令（通过 Function Calling）。
    - 支持孩子手写备注与笔迹序列化。
- [ ] **作业本 (Workbook)**：
    - 米黄横线背景，支持题目展示与答题区域划分。
    - 集成批改功能，AI 可直接在作业本上圈划。
- [ ] **笔记本 (Notebook)**：
    - 白色自由绘制区域。
    - AI 自动整理笔记内容，支持分类标签。

## 3. 核心 Workflow (教学逻辑)
- [ ] **回答问题 Workflow**：提问 -> AI 引导 -> 黑板演示。
- [ ] **按需出题 Workflow**：支持考前复习、错题巩固、模拟考试等多种出题场景。
- [ ] **补习/预习 Workflow**：AI 主动引导课程进度。

## 4. 家长端与系统完善
- [ ] **家长端功能**：查看学习报告、发起抽查、设置学习目标。
- [ ] **UI/UX 美化**：统一插画风格，优化手写笔迹的平滑度，增加组件切换动画。
- [ ] **性能与适配**：优化笔迹渲染性能，确保在平板和手机上的响应式体验。

## 5. 验证与交付
- [ ] **全路径测试**：从提问到生成黑板演示的完整闭环测试。
- [ ] **部署准备**：打包 APK/iOS，配置环境。
